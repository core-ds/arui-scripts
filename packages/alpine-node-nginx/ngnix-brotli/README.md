# Nginx-brotli module

Это модуль для Nginx, который добавляет поддержку Brotli сжатия.

Основан на [оригинальной реализации от Google](https://github.com/google/ngx_brotli/tree/master).

Filter модуль скопирован без изменений.

Static модуль модифицирован для поддержки работы с [Compression-Dictionary-Transport](https://developer.mozilla.org/en-US/docs/Web/HTTP/Guides/Compression_dictionary_transport).

Алгоритм работы браузеров с shared-dictionary:

-  Пользователь загружает ресурс. В заголовке `Use-As-Dictionary` ресурс помечается как "словарь" для будущих версий этого ресурса.
В заголовке содержится маска ресурсов, для которых его надо использовать как словарь. Также есть возможность указать id для этого словаря.
-  При последующих запросах за ресурсами, попадающими под маску, браузер будет отправлять заголовок `Available-Dictionary`, `Dictionary-ID`, если для словаря был задан id, а также в `Accept-Encoding` будет добавляться `dcb`.
-  Сервер может на такие запросы ответить архивом, сжатым с использованием первой версии ресурса.

Модуль предполагает, что кроме статических версий br контента на диске будут находиться версии, сжатые словарем.
Для того чтобы это "просто работало", предполагается, что все имена сжатых словарем файлов имеют структуру типа `{оригинальное имя файла}.{id словаря}.dcb`.

- `{оригинальное имя файла}` - имя сжатого файла вместе с его расширением.
- `{id словаря}` - то значение, которое браузер будет передавать в заголовке `Dictionary-ID`.

Для корректной простановки заголовка `Use-As-Dictionary` имена файлов должны иметь структуру типа `{стабильное имя}.{хэш}.{расширение}`.

- `{стабильное имя}` - имя файла, которое будет оставаться одинаковым для всех сборок приложения, например, `main`, `vendor`.
- `{хэш}` - "идентификатор" этого файла. Будет использоваться как id словаря в заголовке `Use-As-Dictionary`.
- `{расширение}` - расширение файла (`js`, `css` и так далее).

По умолчанию arui-scripts при сборке генерирует файлы именно в этом формате.

#### Основная логика работы:
1. **Проверка условий для обработки запроса:**
    - Метод запроса должен быть `GET` или `HEAD`.
    - URL не должен заканчиваться на `/` (запрос к файлу, а не директории).
    - Если модуль `brotli_static` выключен (`off`), обработка прерывается.
    - Если клиент не поддерживает сжатие Brotli (`accept-encoding: br`), обработка также прекращается.

2. **Попытка отдать DCB-файл (если поддерживается клиентом):**
    - Проверяет наличие заголовков `accept-encoding: dcb`, `dictionary-id` и `available-dictionary`.
    - Сравнивает сигнатуру словаря из запроса с сигнатурой в DCB-файле.
    - Если проверки пройдены, отдаёт содержимое DCB-файла.

3. **Попытка отдать Brotli-сжатый файл (`*.br`):**
    - Если DCB не был отдан, ищет рядом с исходным файлом его Brotli-версию (с расширением `.br`).
    - При наличии такого файла — отдаёт его с заголовком `Content-Encoding: br`.

4. **Дополнительные действия:**
    - Устанавливает заголовок `Use-As-Dictionary`, если запрошенный файл соответствует шаблону для использования в качестве словаря (например, `stableName.hash.ext`).
