# @alfalab/brotli-dcb-builder

## Общее описание

Утилита предназначена для сжатия файлов с использованием предопределённых словарей. Программа анализирует файлы во входной директории,
находит соответствующие словари по определённому шаблону имени и выполняет Brotli-сжатие с использованием словарей.

Архив пишется в формате `.dcb`, подходящем для работы с [Compression Dictionary Transport](https://developer.mozilla.org/en-US/docs/Web/HTTP/Guides/Compression_dictionary_transport).

## Параметры запуска

```bash
npx @alfalab/brotli-dcb-builder --input <path> --output <path> --dictionary <path>
```

| Параметр       | Описание                                                                      |
|----------------|-------------------------------------------------------------------------------|
| `--input`      | Путь к директории с файлами, которые необходимо сжать (обязательный)          |
| `--output`     | Путь к директории для сохранения полученных архивов (обязательный)            |
| `--dictionary` | Путь к директории со словарями для оптимизации сжатия (обязательный)          |
| `--extensions` | Опциональный параметр, переопределяет сжимаемые файлы. По умолчанию `js, css` |

## Алгоритм работы

1. **Чтение входных файлов**:
    - Утилита сканирует директорию `input` и получает список всех файлов для обработки.

2. **Поиск соответствующих словарей**:
    - Для каждого файла из `input` ищется подходящий словарь в директории `dictionary`.
    - Сопоставление происходит по шаблону имени: `{stable_name}.{hash}.{extension}`.
    - Файлы считаются соответствующими, если у них совпадают:
        - `stable_name` (стабильное имя)
        - `extension` (расширение файла)
    - Часть `hash` в имени файла игнорируется при сопоставлении.

3. **Обработка файлов**:
    - Для каждого входного файла:
        - Если найден соответствующий словарь, выполняется сжатие с использованием этого словаря.
        - Если словарь не найден, ничего не делается.
    - Результаты сохраняются в директорию `output`.

4. **Формат выходных файлов**:
    - Имена выходных файлов сохраняют оригинальные имена, добавляя в конце `.{hash}.dcb`, где `{hash}` - хэш из имени словаря.

## Примеры использования

Базовый пример:
```bash
npx @alfalab/brotli-dcb-builder --input ./v2 --output ./archives --dictionary ./v1
```

Пример структуры файлов:
```
v1/
  webpack-assets.json
  main.abc123.js
  vendor.xyz456.css
  unknown.321cba.css

v2/
  webpack-assets.json
  main.def789.js
  vendor.789def.css
  other_data.654fed.dat

output/
  main.def789.js.abc123.dcb
  vendor.789def.css.xyz456.dcb
```
