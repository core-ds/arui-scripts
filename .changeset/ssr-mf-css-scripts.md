Per-expose CSS для module-federation модулей теперь атрибутируется в манифест сборки.

Новый плагин `AttributeModuleCssPlugin` обходит граф чанков от каждого экспоуза
(`configs.modules.exposes`), собирает css чанк-группы экспоуза и её синхронных частей и
записывает их в запись модуля манифеста: `{ mode: 'default', js: 'remoteEntry.js', css: [...] }`.
Раньше css default-модулей оставался только в безымянной записи и не был привязан к модулю,
из-за чего хост-сервер не мог отдать стили модуля при SSR (модуль отрисовывался без стилей
до загрузки на клиенте). Css асинхронных (лениво загружаемых) чанков внутри экспоуза
сознательно не атрибутируется. Изменение аддитивно: для сборок без модулей и для не-SSR
потребления манифест остаётся прежним.

Подробнее: [SSR-спецификация модулей, раздел 10](../packages/arui-scripts-modules/docs/ssr-spec.md#10-addendum-css-delivery-for-module-federation-modules).
